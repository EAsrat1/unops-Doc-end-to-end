    Production Resource Management :

    Dynamic Scaling of Moodle Application Servers:

    During peak periods, all 7 to 10 application servers per cluster will remain active to handle high user loads. During idle or non-peak times, application servers will be scaled down to 2 servers per cluster, reducing resource consumption while maintaining service availability.

    Leveraging Auto-Scaling Services:

    If Ethio-Telecom offers an auto-scaling service, it will be integrated with the Elastic Load Balancer (ELB) to dynamically allocate and decommission application servers based on real-time demand. In the absence of an auto-scaling feature, the scaling process will be managed manually or through orchestration tools like Ansible, ensuring efficient deallocation and decommissioning of resources.

    Resource Decommissioning:

    Excess Moodle application servers will be shut down and deallocated automatically or manually during non-peak times, leveraging Ansible for seamless scaling.

    Elastic Load Balancer and WAF Flexibility:

    The ELB will maintain load balancing with auto-failover capabilities, while WAF instances will scale to handle concurrent connections as needed.

    Caching and Database Services:

    Caching services and database instances will maintain consistent performance, with caching clusters scaled based on the active application server count.

    Ansible Server and Jump Server Utilization:

    The Ansible Server will manage configuration and deployment tasks, while the Jump Server will ensure secure administrative access, both running at minimal capacity during idle times. Outcome: This cost optimization strategy will enable the environment to dynamically adapt to workload changes, minimizing costs while maintaining performance and availability. The integration of Ethio-Telecom's auto-scaling service (if available) will further automate resource management, contributing to operational efficiency and reducing the total cost of ownership (TCO).
